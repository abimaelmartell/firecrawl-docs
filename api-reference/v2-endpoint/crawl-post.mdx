## What's New in v2

### Natural Language Crawler Configuration
The v2 Crawl API introduces a powerful `prompt` parameter for configuring crawler options using natural language:

```json
{
  "url": "https://example.com",
  "prompt": "Crawl only the blog posts and documentation pages, ignore marketing and pricing pages"
}
```

The prompt automatically generates crawler options (excludePaths, includePaths, etc.), though explicitly set parameters will override the generated equivalents.

### Enhanced Sitemap Handling
**v1**: Used `ignoreSitemap` boolean parameter
**v2**: Uses `sitemap` enum with three modes:
- `"include"` (default): Use sitemap along with other discovery methods
- `"skip"`: Ignore the website sitemap completely
- `"only"`: Only crawl URLs found in the sitemap

### Advanced Crawling Parameters
**New in v2:**
- `crawlEntireDomain`: Allows crawling sibling/parent URLs, not just child paths
- `maxDiscoveryDepth`: Controls crawl depth based on discovery order

**Removed from v2:**
- `maxDepth`: No longer available (use `maxDiscoveryDepth` instead)
- `allowBackwardLinks`: Deprecated in favor of `crawlEntireDomain`

### Enhanced Discovery Control
```json
{
  "url": "https://example.com/features",
  "crawlEntireDomain": true,
  "maxDiscoveryDepth": 2,
  "sitemap": "include"
}
```
